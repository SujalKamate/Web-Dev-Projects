Educational Resources & Cache Theory

Cache Mapping Techniques

Main memory block sizes are larger than cache capacity. Mapping strategies resolve where blocks sit in the cache:

  • Direct Mapping: Each RAM block maps to one designated cache line slot. Easy to design, but prone to **conflict thrashing** if multiple active memory blocks share the same cache index.
  • Fully Associative: Any RAM block can sit in any cache slot. Eliminates conflict misses, but requires expensive parallel lookup comparators to scan every tag simultaneously.
  • Set Associative: Compromise mapping. The cache has sets of $N$ lines. A RAM block maps to a designated set, but can sit in any line within that set.

Eviction Replacement Policies

When a new block is loaded into an associative cache set and all slots are full, an existing block must be evicted:

  • LRU (Least Recently Used): Tracks access history and discards the block that hasn't been accessed for the longest period.
  • FIFO (First-In First-Out): Evicts the block that has been residing inside the cache for the longest time, regardless of usage frequency.
  • LFU (Least Frequently Used): Evicts the block with the lowest access count inside the current cache sequence.

Cache Write Policies

Determines how the CPU writes data back to main memory:

  • Write-Through: Updates both the cache block and the main memory block immediately. Slow write performance, but memory data matches cache at all times.
  • Write-Back: Updates only the cache block. A **Dirty Bit** is set to `1`. The actual memory update is deferred until that cache line is evicted, offering fast write times.
