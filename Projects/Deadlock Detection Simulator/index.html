Educational Resources & Theory

What is a Deadlock?

A **deadlock** occurs in an operating system when a set of processes are blocked because each process is holding a resource and waiting for another resource held by some other process in the set. For a deadlock to happen, four conditions (Coffman conditions) must hold simultaneously:

  • Mutual Exclusion: At least one resource must be held in a non-sharable mode.
  • Hold & Wait: A process must hold resource allocations while waiting for requested resources.
  • No Preemption: Resources cannot be forcibly taken from a process holding them.
  • Circular Wait: A closed chain of processes exists, where each process holds resources requested by the next.

Resource Allocation Graph (RAG)

Deadlocks can be described precisely using a directed graph called a **Resource Allocation Graph (RAG)**:

  • Nodes are divided into Processes (circles) and Resources (rectangles).
  • Request Edge ($P_i \to R_j$): Process $P_i$ has requested resource $R_j$ and is currently waiting.
  • Allocation Edge ($R_j \to P_i$): An instance of resource $R_j$ has been allocated to process $P_i$.
  • Cycle Rule: If the graph has no cycles, the system is not deadlocked. If a cycle exists:
    • If all resources have single instances, a cycle implies a deadlock.
    • If resources have multiple instances, a cycle does not necessarily mean a deadlock.

Banker's Detection Algorithm

For multiple-instance resources, we check for deadlocks by trying to find a "safe execution sequence". The algorithm maintains:

  1. Work: Vector of size $m$ (available resources), initialized to Available.
  2. Finish: Boolean array of size $n$, initialized to `false` for processes with non-zero allocations, and `true` for empty ones.
  3. Process Sweep: Find process $i$ such that `Finish[i] === false` and $\text{Request}_i \le \text{Work}$.
  4. Release: If found, set `Work = Work + Allocation` and `Finish[i] = true`, then repeat.
  5. Evaluation: If all finish, system is safe. If any process remains unfinished (`Finish[i] === false`), the system is **deadlocked**, and those unfinished processes form the deadlocked set.
